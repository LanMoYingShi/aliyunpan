# BoxPlayer v5.0.2

> 发布主题：115 网盘视频播放修复、原画字幕流支持、Release 发布链路调整。

## 发布摘要

v5.0.2 是基于 v5.0.1 的增量修复版本。v5.0.1 已经发布了光鸭云盘、多网盘工具、115 云下载、AI tool call 和多网盘整理/扫描能力；本版本重点修复 115 网盘播放体验中的两个高优先级问题：转码视频播放失败，以及原画播放无法正确使用字幕流。

同时，本版本继续完善自动发布链路。由于 GitHub macOS 公证凭证仍需本地验证，CI 暂时关闭 macOS 自动打包，仅保留 Windows/Linux 自动发布。

## 重点修复

- 修复 115 网盘播放转码视频失败问题：补齐 HLS/转码播放地址的媒体类型识别，避免内置网页播放器提示“不支持当前媒体类型”。
- 修复 115 网盘原画不支持字幕流问题：新增 115 字幕接口解析，支持带鉴权 header 的字幕流通过本地代理加载。
- 修复 115 网盘默认清晰度设置为 1080P 时仍进入 4K 的问题，播放器会优先匹配用户在设置中选择的默认清晰度。
- 修复 115 原画使用 MPV 播放时内嵌字幕没有自动启用的问题；当没有外部字幕且 MPV 未选择字幕轨时，会自动选择第一条内嵌字幕轨。
- 修复天翼云盘登录二维码在深色/透明背景下显示不稳定的问题。

## 播放与字幕

- 115 转码播放现在会带上更准确的 video type，播放器能正确区分普通直链、HLS 转码流和原画播放。
- 115 字幕列表会映射为播放器可识别的字幕源，并复用播放接口返回的必要请求头。
- 网页播放器加载字幕时，如果字幕源需要鉴权 header，会走本地 subtitle proxy，避免浏览器直接请求字幕地址时丢失鉴权信息。
- MPV 内嵌播放界面补齐字幕轨自动选择逻辑，减少原画播放“有字幕但默认不显示”的情况。

## 发布链路

- GitHub Actions macOS 打包接入 `CSC_LINK`、`CSC_KEY_PASSWORD`、`APPLE_ID`、`APPLE_PASSWORD` 和 `APPLE_TEAM_ID`。
- GitHub Actions 暂时关闭 macOS 自动打包，待本地验证 Developer ID 签名与 Apple notarization 后再恢复。
- Release 文档继续使用 `docs/releases/v5.0.2.md` 作为 GitHub Release Notes 来源。
- 私有 Client ID、Client Secret、API URL 仍通过 `.env.local` / GitHub Secrets 生成 `src/secrets.generated.ts`，不会提交到公开仓库。

## 相对 v5.0.1 的说明

v5.0.1 已发布以下大功能，本版本不重复作为新增能力宣传：

- 光鸭云盘接入。
- 多网盘空目录、大文件、重复文件扫描。
- 移动整理、媒体整理和插件工具重组。
- 115 云下载与多网盘云下载入口调整。
- AI Agent 感知多网盘工具能力。
- 删除/回收站能力边界收紧。

v5.0.2 的主要价值是把 v5.0.1 后用户反馈的 115 播放、字幕和 macOS 安装问题收口。

## 升级建议

- 使用 115 网盘内置播放器、1080P/720P 转码播放或原画字幕播放的用户建议升级。
- macOS 包待本地签名/公证验证后再发布；当前自动 Release 先提供 Windows/Linux 产物。
- 使用光鸭云盘和多网盘工具的用户，如果已经升级到 v5.0.1，也建议继续升级到 v5.0.2 获取播放与发布包修复。

## 已知限制

- 不同网盘的转码、字幕和原画能力仍受官方接口、会员状态和账号权限影响。
- 115 字幕接口如果返回空列表或官方未提供字幕地址，播放器不会自行生成字幕。
- macOS 签名与公证依赖 Apple Developer 证书、App 专用密码和 Team ID；当前先在本地验证，CI 暂不发布 macOS 包。
